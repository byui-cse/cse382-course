Task 1

Use lazy computation to do some interesting mathematics of your choice using numbers greater than 64 bits. Use a big-number library for your language, if needed.

Task 2

  1. Write pseudocode for a stream that can be access by multiple, concurrently running functions. The stream should be a simple, alpha-numeric, not numeric, counter. Such a stream could be used to provide unique id's for rows in massive data permutations. A good way to do this could be using Heap's Algorithm , not a heap data structure, to generate permutations.

    To aid you in this, here is functional style pseudocode for Heap's Algorithm. It has two sub-algorithms used to make it easier to read. The third is an algorithm for swapping elements in a functional list. The complexity of the swapping sub-algorithm is \(\mathcal{O}(n)\) assuming the split BIF for your language is also \(\mathcal{O}(n)\).

    \[ \begin{align*} &\text{%%Heap's algorithm for generating a list of permutations}\\ heaps\_algo\text{ }&::\text{ }Int\text{ }[a]\text{ }[a]\rightarrow [a]\\ \\ heaps\_algo\text{ }&::\text{ }1\text{ }permutation\text{ }accum\rightarrow\\ &[permutation]\doubleplus accum;\\ heaps\_algo\text{ }&::\text{ }k\text{ }permutation\text{ }accum\rightarrow\\ &perms\_for\text{ }::\text{ }k\text{ }1\text{ }(heaps\_algo\text{ }::\text{ }k-1\text{ }permutation\text{ }accum). \\ \\ &\text{%%the perms_for sub-algorithm}\\ perms\_for\text{ }&::\text{ }Int\text{ }Int\text{ }[a]\rightarrow [a]\\ \\ perms\_for\text{ }&::\text{ }k\text{ }i\text{ }accum\text{ }when\text{ }k = i\rightarrow\\ &accum;\\ perms\_for\text{ }&::\text{ }k\text{ }i\text{ }accum\text{ }when\text{ }k\pmod 2 = 0\rightarrow\\ &[h\mid \_] = accum\\ &perms\_for\text{ }::\text{ }k\text{ }(i+1)\text{ }(heaps\_algo\text{ }::\text{ }k-1\text{ }(swap\text{ }::\text{ }i,k,h)\text{ }accum)\\ perms\_for\text{ }&::\text{ }k\text{ }i\text{ }accum\rightarrow\\ &[h\mid \_] = accum\\ &perms\_for\text{ }::\text{ }k\text{ }(i+1)\text{ }(heaps\_algo\text{ }::\text{ }k-1\text{ }(swap\text{ }::\text{ }1,k,h)\text{ }accum)\\ \\ \\ &\text{%%the swap sub-algorithm}\\ swap\text{ }&::\text{ }a\text{ }a\text{ }[a]\rightarrow [a]\\ \\ swap\text{ }&::\text{ }a\text{ }b\text{ }list\rightarrow\\ &lesser = min\text{ }::\text{ }a\text{ }b\\ &greater = max\text{ }::\text{ }a\text{ }b\\ &(head,[elem1\mid t1]) = split\text{ }::\text{ }(lesser - 1)\text{ }list\\ &(mid,[elem2\mid end]) = split\text{ }::\text{ }(greater-lesser-1)\text{ }t1\\ &head\doubleplus elem2\doubleplus mid\doubleplus elem1\doubleplus end. \end{align*} \]

    Hint

    You can listen for a request in any function called by your process.

  2. Use a modified version of Heap's Algorithm to write code in you language that streams single permutations using lazy, one-at-a-time generation per request.
  3. Test your counter serially.
  4. Test your counter concurrently.

Task 3

  1. Write pseudocode for a stream that uses a spigot algorithm to produce a stream of digits of pi.
  2. Write the code for this stream in your language.
  3. Test your stream.

Task 3

  1. Write pseudocode for a stream that has as a parameter the HTML for a web page and provides a stream of all the links in the page.
  2. Write the code for this stream in your language.
  3. Test your stream using a function that retrieves BYU-Idaho's main page.

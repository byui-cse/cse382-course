CSE 382: Week 06

Tries

Your Tasks

Task 1

Create pseudocode for and, using your language, write as a monoid the following list of monoidal functions.

  1. append
  2. difference
  3. merge - intermingle the elements of lists
  4. id - a function, with no parameters, that returns the identity for these list functions
Write code that exercises the functions you've written.

Task 2

Consider these behaviors that work on lists. Could they be added the monoid from Task 1? Why or why not?

  • is_element_of
  • intersection
  • symmetric_difference
  • create_sublist_where
  • is_sublist_of
  • Assuming the elements of the list are numeric, a pair of functions called divide_all_by and add_to_all

    • Task 3

    Write, in English, a description of the composition rule for the monoid you've been creating.

    Task 4

    Consider the maybe monad. In your language, write

    1. the\(monad\_chain\) function,
    2. the \(maybe\_bind\) function,
    3. the \(from_user\) function to get the URL from the user,
    4. the \(get\) function to retrieve data,
    5. the \(save\) function to save the data retrieved to disk, and
    6. code that exercises your code.

    Task 5

    Brainstorm and describe, in English, a monad, not the maybe monad, that you could create and use. Please realize that monads are an architectural pattern, not something you would use as a pattern for a single function.

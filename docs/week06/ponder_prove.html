CSE 382: Week 06

Tries

Your Tasks

Task 1

Using the pseudocode learned in this class, write pseudocode for a monoid consisting of the following monoidal functions working on 2-tuples consisting of two lists.

  1. append
  2. difference
  3. merge - intermingle the elements of lists
  4. id - a function, with no parameters, that returns the identity for these list functions

Task 2

Implement these same functions in the tasks.erl found in the monad_monoid.zip file so they pass the unit tests in that file.

Task 3

Consider these behaviors that work on lists. Could they be added the monoid from Task 1? Why or why not?

  • is_element_of
  • intersection
  • symmetric_difference
  • create_sublist_where
  • is_sublist_of
  • Assuming the elements of the list are numeric, a pair of functions called divide_all_by and add_to_all

    • Task 4

    Write, in English, a description of the composition rule for the monoid you've been creating.

    Task 5

    Consider the maybe monad. In your language, write code for

    1. the \(monad\_chain\) function,
    2. the \(maybe\_bind\) function,
    and pass all of the unit tests for these functions.

    Task 6

    Brainstorm and describe, in English, a monad, not the maybe monad or one discussed in class, that you could create and use. Please remember that monads are an architectural pattern, not something you would use as a pattern for a single function.

    Review

    After completing the previous tasks:

    1. Review one of the many possible solutions for this task,
    2. Compare and contrast your solutions with the possible solutions, and
    3. Make note of any improvements you could make for future tasks.

The Tree Data Structure

Trees follow the monad pattern. There is a group of data organized in some way, a series of functors that operate on that group, and each of the rules for monad is met. That's why you'll hear, "A tree is a monad" used colloquially. The more correct language would be, "Well made trees and the functions that operate on them follow the monad pattern." That's a little long. Maybe we should stick with "A tree is a monad" for ease of speech and just remember what we actually mean.

For this course the functors we'll focus on are\(add,contains,\text{ and }replace). A fully fledged set of functors wuld include other behaviors.

All types of trees consist of nodes. These nodes can have more than one \(next\). The lists you learned about in week 02 are a subset of trees where there is only one \(next\), a unary tree.

Binary Tree

Binary trees have two \(nexts\).

For \(n\in\{-1,0,1\}\), \[\begin{align*}add\text{ }&::\text{ }node,a,(comp\text{ }::\text{ }a,b\rightarrow n)\rightarrow root\_node,\\ contains\text{ }&::\text{ }a,(comp\text{ }::\text{ }a,b\rightarrow n)\rightarrow Boolean\\ replace\text{ }&::\text{ }a,b,(comp\text{ }::\text{ }a,b\rightarrow n)\rightarrow root\_node \end{align*}\]

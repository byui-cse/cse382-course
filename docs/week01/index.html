Patterns and Data Structures and Algorithms...Oh My!

These big three parts of programming, regardless of the language you choose, are related. An Algorithm is a pattern consisting of a series of steps that is known to accomplish a specific task with a known efficiency. A Data Structure is a pattern that lays out how to store, retrieve, and manipulate data with a known efficiency. This connection between patterns, data structures, and algorithms means that we can, and will, use the more abstract word pattern to describe both of the other two types, reserving the more detailed words, algorithm and data structure, when they illuminate the topic.

As time goes by, you will realize that algorithms and data structures interact as well. Useful algorithms use or produce one or more data structures, and data structures use one or more algorithms. Such is the complexity and recursiveness of life. Nothing is actually independent of everything else. Instead, everything is dependent, directly or indirectly, on everything else.
